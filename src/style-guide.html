Guidelines

  • Build pages that are usable on all devices
    How To Meet This Principle
    • Start building your page from a mobile perspective, ordering elements by priority, and rearrange those content pieces for each device up to desktop
    • Use media queries to apply specific styling to different viewports
    • Take advantage of the grid system, which has predefined behaviors for each device
    • Test your websites on different devices and different resolutions to make sure the page does not "break" at any point
    Further Reading
    • Mobile First
    • Responsive Web Design
  • Keep website and page structure simple and easy to navigate
    How To Meet This Principle
    • Limit global navigation items between 4 to 7 items
    • Limit the number of child pages to an element between 5 and 9
    • Combine several pages of similar content into sections of content on a single page
    • Every page should meet the end goal of at least one target audience
    • Complete information architecture building exercises
    Further Reading
    • Usability Guidelines 16:7 - Display Only Necessary Information
    • User-Centered Design and Web Development
    • 30 Usability Issues To Be Aware Of
  • Facilitate accessibility through good coding and design practices
    How To Meet This Principle
    • Use an HTML5 validator
    • Only use tags for their intended purpose, and not for the appearance that tag provides
    • Do not use inline styling
    • Provide text alternatives to non-text elements
    • Follow accessibility guidelines
    Further Reading
    • Usability Guidelines - Accessibility
    • HTML Validator
    • HTML Tutorial - (HTML5 Compliant)
  • Keep the page light weight
    How To Meet This Principle
    • Concatenate and minify HTML and Javascript
    • Place any extraneous javascript before the closing body tag
    • Limit the number of images on a page
    • Compress images used on a page
    • Aim to make pages less than 500kb in size
    • Evaluate your page with tools like PageSpeed and YSlow

Grid System

You can utilize the grid system that is built into the framework to arrange content in various ways that may better suit the content you are displaying.

The grid system consists of 12 fluid columns fit into a container. To use the grid, create a row and then create the columns you desire within the row. You can use the preset span classes (.span1, .span2, .span3, etc) to make an element span multiple columns. The total number of columns spanned per row should be 12.

On devices with a display resolution less than 690px, each grid column spans the entire width of the screen.

Creating Rows and Columns

Images

Images should be used sparingly on the page, and only when they compliment the content. Images that are poor quality, obvious stock images, excessively large, or distasteful are not to be used.

Images should not be used to display text content, only photos and figures. Any text required should be placed over the image in HTML so it remains accessible and searchable.

Image Sizes

Images change dimensions but maintain aspect ratios across devices. Here are some common image ratios (width:height) used throughout the University websites.

  • Full page feature: 10:3 (e.g. 1680 x 504)
  • Faculty feature / News story images: 16:9 (e.g. 780 x 440)
  • Thumbnail images: 1:1 (e.g. 80 x 80)
  • Wide thumbnail images: 16:9 (e.g 156 x 88)

Image Classes

The following classes can be applied to a div or an image to apply various behaviors:

  • pull-left: Floats the div or image to the left side of the container.
  • pull-right: Floats the div or image to the right side of the container.
  • rounded: adds rounded corners to the image / div.
  • img-circle: turns the image into a circle.

Video

The video component allows you to display a thumbnail image with a play icon and caption. When the user clicks the thumbnail, the video will appear in a modal window. If the user is on a mobile device, they will be directed to the mobile friendly page for the video. You have the option of providing a thumbnail for the video, or automatically obtaining one from the provider. Only YouTube and Vimeo are supported currently.

Carousel

A carousel contains several slides that the user can navigate through. Slides can contain an image with a caption, video, or HTML content. The carousel should be used to display content with a high priority to the user, such as highlighting news stories or upcoming events. The carousel should not be used to display static images.

Accordion

The accordion is used to display large amounts of nested links. The user has the choice of clicking the top level link, or choosing to expand the top level element and browse the links below it. Because the accordion has such a large number of links, the links should be ordered alphabetically so the user can easily find what they are looking for. If the number of top level categories is less than 10, the categorized link bucket should be used instead.

The cateogirzed link bucket is similar to the accordion, but displays links in a horizontal manner as opposed to vertically. The categorized link bucket is also filterable by typing a query into the input field labelled "filter links". Links can optionally be sorted by popularity (from analytics) instead of alphabetically. The number of categories (on the left) should not exceed 10.

Institutional Home

The institutional home page does not exist within Sitecore. It is a static HTML page that is updated manually by a member of the University Digital Strategy team.

The ualberta.ca homepage follows a three column layout. The feature area spans the whole page while the why ualberta space spans the width of the 3 columns. The explore bar takes up the width of the primary and secondary columns, with the sidebar on the right. The news appears in the primary content area under the explore bar. The campus notices and colloquy blog feeds are displayed in the secondary content space next to the news. Lastly, the twitter feed and social media links are displayed in the sidebar.
